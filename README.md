# 🐱Сегментация изображений

## 📌 Описание проекта и датасета

В качестве обучающих данных используется специализированный строительный датасет, содержащий снимки объектов на этапе внутренней отделки и ремонта с соответствующими попиксельными масками разметки.

* **Размер изображений:** 256 x 192 пикселей.
* **Объем выборки:** 1900 обучающих снимков (`train`) и 100 проверочных снимков (`val`).
* **Исходная разметка:** 16 детализированных классов (пол, потолок, стены, колонны, проемы, окна, проводка, люди, инвентарь и т.д.).

---

## 📂 Структура репозитория и выполняемые задачи

Репозиторий разделен на 3 ключевые части (соответствующие файлам `l10z1.ipynb`, `l10z2.ipynb`, `l10z3.ipynb`), каждая из которых решает свою архитектурную или аналитическую задачу:

### 1. Архитектура U-Net (Макро-категории) — `l10z1.ipynb`
* **Задача:** Реализация базовой архитектуры U-Net для укрупненной сегментации.
* **Группировка классов (5 категорий):**
  * `Класс 0` — Несущие элементы (Стены, колонны, балки)
  * `Класс 1` — Перекрытия (Пол, потолок)
  * `Класс 2` — Проемы, двери и окна
  * `Класс 3` — Коммуникации и внутренние объекты (Инвентарь, батареи, проводка, лампы)
  * `Класс 4` — Внешний мир и люди
* **Особенности:** Использование классических блоков `Conv2D` -> `BatchNormalization` -> `MaxPooling2D` в энкодере и `Conv2DTranspose` со сквозными связями (`Skip Connections`) в декодере.

### 2. Полноразмерный U-Net (7 классов) — `l10z2.ipynb`
* **Задача:** Повышение детализации сегментации и масштабирование сети до 7 независимых классов.
* **Группировка классов:**
  * `0`: Пол | `1`: Потолок | `2`: Стены
  * `3`: Проемы, двери, окна
  * `4`: Колонны, перила, лестницы
  * `5`: Инвентарь
  * `6`: Лампы, провода, балки, внешний мир, батареи, люди
* **Особенности:** Расширенная конфигурация фильтров, детальный попиксельный анализ точности на валидационной выборке.

### 3. Пирамидальный PSPNet (Advanced) — `l10z3.ipynb`
* **Задача:** Реализация современной архитектуры **PSPNet (Pyramid Scene Parsing Network)** для извлечения глобального контекста изображения.
* **Архитектурные особенности:**
  * В качестве бэкбона выступает глубокий сверточный энкодер, снижающий размер карты признаков до `32 x 24`.
  * **Модуль пирамидального пулинга (Pyramid Pooling Module):** Формирование четырех масштабов подрешеток (бин-размеры `1x1`, `2x2`, `3x3`, `6x6`) с помощью `AveragePooling2D`.
  * Уменьшение каналов сверткой `1x1` и последующая билинейная интерполяция (`tf.image.resize`) обратно к исходному масштабу карты признаков.
  * Конкатенация глобальных и локальных признаков для точной локализации мелких объектов (например, проводов и светильников).
